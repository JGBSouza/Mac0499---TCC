Automatização do fluxo de submissões de patches para o Kernel Linux através do kworkflow

Aluno: João Guilherme Barbosa de Souza

Orientador: Paulo Meirelles

Coorientador: David Tadokoro

Resumo:

O Kernel Linux é hoje um dos maiores projetos de software livre existentes, contando com a colaboração de desenvolvedores em todo o mundo por mais de 30 anos. Em ordem de gerir o grande fluxo de submissões, o ecossistema GNU/Linux conta com um modelo de desenvolvimento único empregado através de correio eletrônico e listas de e-mail que encapsulam todas as alterações propostas por um possı́vel contribuidor.

Para dar suporte a esse modelo de desenvolvimento, diversos softwares se propõem a melhorar a experiência dos usuários, automatizando processos complicados que são corriqueiros e prejudicam o fluxo de contribuição. Além disso, esses processos também podem representar uma grande barreira para novos colaboradores menos experientes.

É nesse contexto que surge também o Kernel workflow (kw), como uma plataforma que busca unificar essas soluções e aprimorar o processo de desenvolvimento para o Kernel Linux. Como software livre, assim como o Kernel, o kw é desenvolvido como software livre, tendo seu grupo de desenvolvimento composto em geral por colaboradores voluntários ou através de projetos científicos, bem como é o caso do presente trabalho

Objetivos

Dessa maneira, esse projeto tem por objetivo se focar especificamente em realizar automatizações em torno do fluxo de submissões de patches para o Kernel Linux, adicionando novas ferramentas para incrementar essa etapa e permitir que os desenvolvedores contribuindo para o Kernel Linux tenham um ambiente integrado e isolado, acessivel através do terminal, por linha de comando, no qual podem visualizar a submissão dos seus patches, controlar seus estados, checar reviews entre outras funcionalidades. De maneira adicional, o trabalho também busca investigar formas de automatizar o fluxo de consultas e revisões dos patches submetidos
